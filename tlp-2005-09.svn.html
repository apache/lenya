History of Changes

Introduction and explanation of symbols

Changes are sorted by "{0}" and thentype chronologically with the most recent at the top. These symbols denote the various action types:=add, =remove, =update

Version 2005-09 ()

1.4

  • [1.4] Renamed publet.xconf to module.xconf Diff Committed by andreas.
  • [1.4] Formating changes of usecase.js. Added basic flow util js which contains io functions that can be used in the custom flow extensions. Activated the util class in the usecase.js. Diff Committed by thorsten.
  • [1.4] Added CustomMetaDataModule to retrieve custom meta data. This fixes bug #36668. Thanks to Felix Roethenbacher (and sorry that I forgot to mention you in my last commit ...) Diff Committed by andreas.
  • [1.4] Added notification module. Diff Committed by andreas.
  • [1.4] Zip source added. Maybe move them into Cocoon at some later time Diff Committed by michi.
  • [1.4] opendocument module started Diff Committed by michi.
  • [1.4] Moved XSLT and homepage schemas to corresponding modules. This required to translate fallback:// rng includes to absolute urls (translate-rng-includes.xsl) so that they can be requested by BXE. Diff Committed by andreas.
  • [1.4] RollbackAction replaced with Rollback usecase Diff Committed by chestnut.
  • [1.4] [minor change] added ASF header Diff Committed by andreas.
  • [1.4] added an outro flowscript matcher. This way it possible to e.g. save files to the file system or do any other action when the form returns to the script. Diff Committed by thorsten.
  • [1.4] The usecase util classes are not usable ATM because of the Rhino bug Diff Committed by thorsten.
  • [1.4] Enhanced *code* documentation of the generic flow object that you can use in custom flow script extensions in usecases. Alternatively one can use the invoking java class by providing bean properties methods (getter/setter methods) in this class and use them in the extensions. Diff Committed by thorsten.
  • [1.4] Simplified the cform definitions for the usecase-fw Diff Committed by thorsten.
  • [1.4] AbstractUsecase: wrap error messages in new list to avoid seeing concurrent modifications Diff Committed by andreas.
  • [1.4] Default Pub: Allow resource types without custom XSLT Diff Committed by andreas.
  • [1.4] Introduced Validator service. The JingValidator is based on Cocoon's JingTransformer (validation block) and allows to use protocols in Relax NG includes. This way, we can use fallback:// to resolve core schemas. Diff Committed by andreas.
  • [1.4] [minor change] removed jing transformer declaration Diff Committed by andreas.
  • [1.4] thanks to Gregor Imboden also the SimpleSiteManager allows deleting documents Diff Committed by michi.
  • [1.4] thanks to Gregor Imboden also meta data is being deleted Diff Committed by michi.
  • [1.4] Add correct URL prefix for relative schema locations. Diff Committed by andreas.
  • [1.4] thanks to a patch by Felix Röthenbacher fixes bug 36539 Diff Committed by chestnut.
  • [1.4] Removed custom Lenya validation code, using Cocoon validation components instead. Diff Committed by andreas.
  • [1.4] Added the proxy to the generic var Diff Committed by thorsten.
  • [1.4] Applied patch for bug #36553 (API change in validation block). Thanks to Felix Roethenbacher. Diff Committed by andreas.
  • [1.4] Getting rid of the hardcoded path. That enables the usage of the composer in any pub that has the compilation source type. Commented out the save action, there is a bug that I am tracking since days in lenya and actually I do not know why it occurs. As workaround for the presentation I recommend to save it to the file system directly and after the presentation keep on debugging. Diff Committed by thorsten.
  • [1.4] lock doc before rollback, unlock after Diff Committed by chestnut.
  • [1.4] added method to get backup entries from RCML-file Diff Committed by chestnut.
  • [1.4] changed checkin at the end of a rollback to make a new version if making a backup Diff Committed by chestnut.

  • [1.4] moved match for modules above usecase match so that modules could be used in usecase views (live usecase might want to use site navigation from sitetree module) Diff Committed by chestnut.
