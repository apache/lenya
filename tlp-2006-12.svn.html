History of Changes

Introduction and explanation of symbols

Changes are sorted by "{0}" and thentype chronologically with the most recent at the top. These symbols denote the various action types:=add, =remove, =update

Version 2006-12 ()

1.4

  • [1.4] Added internal linking functionality, should be standardized with other editors, but it is a start Diff Committed by chestnut.
  • [1.4] Refactoring: extract abstract superclass from DefaultNotifier, send Publish notification to submitter Diff Committed by andreas.
  • [1.4] Added collection module which provides the resource type necessary to make collection test work, register SourceNode as dirty only when output stream is closed Diff Committed by andreas.
  • [1.4] Providing alternative sitetree implementation with better performance. Diff Committed by andreas.
  • [1.4] Improved and extended notification and inbox Diff Committed by andreas.
  • [1.4] links should utilize uuids now Diff Committed by chestnut.
  • [1.4] don't resolve uuid links if rendertype is edit Diff Committed by chestnut.
  • [1.4] Changed resource type menus and 'new document' i18n Diff Committed by andreas.
  • [1.4] Added logger to Identity constructor, added utility method to resolve users with a certain role, use it in Publish usecase for notification Diff Committed by andreas.
  • [1.4] Refactored notification mechanism (more generic now), notify listeners after transactions are completed, added basic inbox functionality (mainly to simplify notification unit testing) Diff Committed by andreas.
  • [1.4] Moved translation functionality up to AbstractNotifier, fixed i18n for notification, don't check if document exists before reading resource type meta data Diff Committed by andreas.
  • [1.4] Don't copy module and resource type declarations to child publications Diff Committed by andreas.
  • [1.4] Removed unnecessary method from SimpleSiteManager, added usecase policies for blog pub Diff Committed by andreas.
  • [1.4] Removed invalid module references from blog publication Diff Committed by andreas.
  • [1.4] Resetting custom loop flow when executing a usecase, otherwise the custom flow 'hangs' Diff Committed by andreas.
  • [1.4] Improved layout of workflow overview Diff Committed by andreas.
  • [1.4] Removed 'subtree' functionality from Publish and Deactivate usecases, see bug 40670 Diff Committed by andreas.
  • [1.4] Preserve ordering of resource type samples Diff Committed by andreas.
  • [1.4] Fixed title i18n for create usecase Diff Committed by andreas.
  • [1.4] Don't re-init the samples in Create.doCheckExecutionConditions() Diff Committed by andreas.
  • [1.4] Set i18n pipelines to caching, close aggregated source validity correctly (improves performance) Diff Committed by andreas.
  • [1.4] Added module.compile target, don't copy java files to classes directory Diff Committed by andreas.
  • [1.4] Don't copy content of default pub to test pub Diff Committed by andreas.
  • [1.4] Performance improvements in URL handling in file policy manager. Diff Committed by andreas.
  • [1.4] Refactoring of services to improve performance Diff Committed by andreas.
  • [1.4] adds missing FCKEditor save button Thanks to Richard Frovarp for the patch resolves Bug 41173 Diff Committed by chestnut.
  • [1.4] Disable upload feature in FCKEditor Thanks to a patch from Richard Frovarp resolves Bug 41172 Diff Committed by chestnut.
  • [1.4] Allow to use extensions instead of regexps in BypassableAccessController, various refactorings to improve performance, e.g. avoid duplicate source resolving Diff Committed by andreas.
  • [1.4] Avoid NPE in FallbackSourceFactory when resolved source is null. Thanks to Richard Frovarp for the patch (although I altered it a bit) Diff Committed by andreas.
  • [1.4] Added notififaction to submit + reject usecases. Thanks to Richard Frovarp for the patch, Diff Committed by andreas.
  • [1.4] Show menu on multi-workflow usecase screen Diff Committed by andreas.
  • [1.4] Use inbox notifier instead of email notifier by default Diff Committed by andreas.
  • [1.4] Refactored and generalized create screens, added some i18n messages Diff Committed by andreas.
  • [1.4] Improved search screen layout and i18n Diff Committed by andreas.
