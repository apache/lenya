Preamble

Wyona CMS is the refactored version of XPS (Extensible Publishing System). XPS is currently used by Switzerland's most famous newspaper Neue Zürcher Zeitung for their interactive edition NZZ Online. The system is being used 7 days a week and 24 hours a day by editors and automatic processes (i.e. nightly generation of the online version of the print edition). It is up and running since October 2, 2000. XPS was Open Source from the beginning (Apache license). Concerning the know-how within the system, it should have become a killer CMS (at least for newspapers), but we made two crucial mistakes: Bad Documentation and a bad installer. We hopefully do a better job this time.

Introduction

Wyona CMS is an Open-Source Content Management and Publishing System written in 100% pure Java. It is based on open standards such as XML and XSLT. One of it's core components is the pipeline processor Cocoon from the Apache Software Foundation.

Wyona is actually just an interpreter of various XMLs, not just user's data, such as for instance articles, press-releases or addresses, but also regarding functionalities, such as for instance access controlling (IML and PML resp. XACML), revision controlling (RCML) or workflow management (WFML). The concept is similar to the one of a "HTML Browser". You should be able to switch the Content Management System at any time. The network is the CMS.

Wyona supports inline WYSIWYG editing, which means users (editors) do not have to learn (or see) XML. This is possible thanks to the support of XML, XSLT and XML Schema on the client side, such is the case for certain browsers (Netscape 6.2 or IE 5.5). But there also exists the "Wyona Administrator" for the more advanced user, where the CMS can be monitored, XML can be edited directly or authentication and authorization can be managed.

You are very welcome to participate in the Wyona CMS project. Thank you very much for your suggestions and contributions.

Download

Regular stable snapshots can be downloaded from the Wyona CMS homepage.

For those living on the bleeding edge, we provide anonymous CVS access to the current development version.

cvs -d :pserver:anonwyonacms@194.191.122.220:/repository checkout wyonacms
or (bash syntax)

export CVSROOT=:pserver:anonwyonacms@194.191.122.220:/repository

cvs checkout wyonacms
